Main challenges

Actually there were none. But considering my extra-curricular activities, there were a few.
- Just recollecting how sketches work and how to connect different ones did take a few minutes to refresh my memory
- Sending the data between Arduino and Processing can be quite tricky at times. If something doesn't work, you usually won't know why until you manually search for the mistakes/missing parts.
- Obviously, Arduino can get quite frequently very annoying, as the uploaded sketch will tell you quite often, that it's busy. As if its time is more precious than mine, daum Arduinos, taking over the world ._.
- Getting to know which is Blue/Red/Green in the RGB LED may be tricky, although straightforward, it ain't color coded, so it's more of a test and trial to get the pins and values right
- How do I send multiple values through serial? There must be a way, probably via an array, but mine did not quite work yet.
- Processing code works somewhat unexpectedly at times. For instance, if the clicked-on value is recorded as a value smaller than 0, Arduino will treat it as some sort of a maximum value. My RGB LED was quite bright, hence my hypothesis.
- The buttons did not always work and I still could not figure out why.
